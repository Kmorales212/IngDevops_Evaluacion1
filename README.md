Estrategia de Ramificación: Trunk-Based Development

Elegimos Trunk based development para este encargo.
A diferencia de gitFlow la estrategia Trunk fomenta la integracion continua al implementar ramas de corta duración. lo que permite apurar el despliegue y evitar conflictos en los merge, en resumen seria la metodologia mas eficiente ya que aumenta la trazabilidad del codigo y agiliza el flujo colaborativo en la nube.
